# 如何贡献与更新

不会 Git 也可以参与。小修正可直接编辑网页，大文件可在课程目录中上传；如果仍不确定，开一个“资料投稿”Issue 交给维护者整理。

## 上传前的 30 秒检查

- [ ] 这是我原创的，或我能指出允许再分发的明确授权；
- [ ] 文件中没有姓名、学号、邮箱、手机号、成绩、手写签名等个人信息；
- [ ] 不是盗版教材、付费内容、未授权课件或正在进行的考核内容；
- [ ] 文件名能看出年份、学期、类型与主题；
- [ ] 我会同步更新该课程的资料清单，而不是只扔一个文件。

不确定能否上传时，先阅读[版权与隐私规范](policies/copyright.md)，只提交描述和公开链接，不要先上传文件。

## 方法一：只用 GitHub 网页（推荐新手）

以“数学分析”资料为例：

1. 登录 GitHub，打开本项目，点击右上角 **Fork**；
2. 进入 `docs/courses/math-analysis/resources/`，按下表选择一个分类目录；
3. 在分类目录中点击 **Add file → Upload files**，上传文件并填写提交说明；
4. 返回 `docs/courses/math-analysis/resources/index.md`，点击铅笔图标，按照本页下方“怎样登记一份资料”新增一行；
5. 打开 `docs/courses/math-analysis/index.md`，按照本页下方“什么时候更新课程攻略”判断需要修改哪些内容；
6. 点击 **Contribute → Open pull request**，在清单中勾选版权与隐私确认项。

| 资料内容 | 放入目录 | 例子 |
| --- | --- | --- |
| 原创课堂或自学笔记 | `notes/` | 分章节笔记、概念辨析 |
| 复习提纲、知识框架 | `review/` | 期末复习提纲、公式索引 |
| 获准公开的往年试卷或题型整理 | `exams/` | 已脱敏原卷、本人原创回忆版 |
| 本人原创题解、勘误或公开授权材料 | `homework/` | 作业勘误、原创题解 |
| 确实无法归入以上类别的合规资料 | `other/` | 课程工具使用说明 |

推荐文件名：`年份-学期-类型-简述.ext`，例如：

```text
2025-秋-笔记-极限与连续.pdf
2024-春-复习-多元微积分.md
2023-秋-试卷-期末-已脱敏.pdf
```

同一文件不要使用“最终版”“最新版”这类会过期的名称。扫描件请先旋转、裁边、压缩并确认文字可读；单文件建议不超过 25 MB。

## 方法二：命令行更新

```bash
git clone https://github.com/<你的账号>/<仓库名>.git
cd <仓库名>
git switch -c add/math-analysis-<资料简称>
```

把资料复制到对应分类后，编辑资源清单，再执行：

```bash
python scripts/check_resources.py
python -m mkdocs build --strict
python scripts/check_site_links.py
git add docs/courses/math-analysis
git commit -m "docs(math-analysis): add <资料简称>"
git push -u origin add/math-analysis-<资料简称>
```

然后在 GitHub 发起 Pull Request。一次 PR 尽量只处理一门课程或一组紧密相关的资料，便于审核和撤回。

## 怎样登记一份资料

上传文件并不等于登记完成。网站只会展示 `resources/index.md` 表格中的内容，因此必须再增加一行 Markdown。

### 第一步：写对文件链接

链接由“分类目录名 + 完整文件名”组成，并且必须与刚上传的文件完全一致。例如文件位于：

```text
docs/courses/math-analysis/resources/notes/2025-秋-笔记-极限与连续.pdf
```

表格中的相对链接就是：

```markdown
[2025-秋-笔记-极限与连续.pdf](notes/2025-秋-笔记-极限与连续.pdf)
```

不要填写电脑上的 `C:\`、`D:\` 路径，也不要只写文件名而漏掉 `notes/` 或 `exams/`。

### 第二步：逐列填写

| 列名 | 应填写什么 | 合格示例 |
| --- | --- | --- |
| 资源 | 可点击的相对链接 | `[文件名.pdf](notes/文件名.pdf)` |
| 类型 | 资料的实际类型 | 笔记、复习、试卷、原创题解 |
| 适用学期/范围 | 学年学期、院系、章节或考试阶段；不知道的部分写“待核验” | `2025 秋 / 第一至三章` |
| 来源或作者 | 谁创作或发布了它 | `匿名贡献者（本人原创）`、`课程组公开发布` |
| 许可/授权 | 为什么本项目有权公开再分发 | `本人原创，CC BY-NC-SA 4.0`、`经原作者书面同意公开` |
| 备注 | 脱敏、答案、扫描质量等补充信息 | `已脱敏；含参考答案` |

原创笔记可以照下面的格式填写：

```markdown
| [2025-秋-笔记-极限与连续.pdf](notes/2025-秋-笔记-极限与连续.pdf) | 笔记 | 2025 秋 / 第一至三章 | 匿名贡献者（本人原创） | 本人原创，CC BY-NC-SA 4.0 | 已脱敏 |
```

获授权试卷的格式如下。以下日期、院系和授权描述只是演示，必须替换成真实信息：

```markdown
| [2025-春-试卷-期末-已脱敏.pdf](exams/2025-春-试卷-期末-已脱敏.pdf) | 试卷 | 2025 春 / 期末 / 计算机学院 | 当学期课程组 | 经课程组于 2025-07-01 书面同意公开，授权记录见 PR #123 | 已脱敏；无答案 |
```

如果没有原卷的再分发授权，可以提交本人原创的题型回忆整理，而不是上传原卷：

```markdown
| [2025-春-期末题型回忆.md](exams/2025-春-期末题型回忆.md) | 试卷回忆 | 2025 春 / 期末 | 匿名贡献者（本人回忆整理） | 本人原创，CC BY-NC-SA 4.0 | 非原卷；题目表述可能不完整 |
```

!!! danger "这些内容不能留空或保留占位文字"
    “请填写真实来源”“请填写授权依据”“群文件”“网上找到的”都不是有效登记。如果无法确认来源或公开再分发许可，请先开 Issue 说明情况，不要上传文件。

如果作者希望匿名，可以写“匿名贡献者”；但维护者仍需能从 PR 或 Issue 记录确认授权。若材料来自公开网页，请链接原页面，并写明页面展示的开放许可。

## 什么时候更新课程攻略

不是每次上传资料都要重写课程介绍。先按下表判断：

| 你掌握的新信息 | 需要修改的位置 |
| --- | --- |
| 只是新增一份文件 | 更新 `resources/index.md`，并在课程页“更新记录”增加一行；其他章节不用改 |
| 核验了课程代码、院系、学分或教材 | 修改“课程档案”对应字段，并注明学期与来源 |
| 核验了某学期考核比例、范围或题型 | 在“考核信息”表格新增一行，不要覆盖旧学期 |
| 有跨学期仍有价值的学习方法 | 在“推荐学习节奏”或“复习框架”补充，并标注适用范围 |
| 发现页面内容已经过期或错误 | 更正原文，并在“更新记录”说明改了什么 |

### 只上传资料时，写一条更新记录

在 `docs/courses/math-analysis/index.md` 最后的“更新记录”表格增加一行：

```markdown
| 2026-07-13 | 收录 2025 春期末试卷 | 匿名贡献者 |
```

日期写提交当天，变更说明要能让读者知道新增或修正了什么。不要只写“更新”“补充资料”。

### 核验了考核信息时，新增历史记录

如果你确实知道当学期的院系、课程代码、考核构成和来源，可以在“考核信息”表格新增一行：

```markdown
| 2025 春 | 计算机学院 / 课程代码请替换 | 平时 30% + 期末 70% | 期末范围请替换 | 当学期教学大纲或教师通知 | 已核验 |
```

示例中的比例和文字不能直接照抄，必须替换为真实信息。只知道部分信息时，未知单元格写“待核验”，不要猜测。

### 补充学习经验时，写清适用范围

把经验放到最相关的章节末尾，并明确这是个人经验还是可核验规则，例如：

```markdown
### 级数部分的学习建议

> 适用范围：2025 春，计算机学院；以下为个人学习经验。

建议为每一种收敛判别法分别整理适用条件、典型正例和反例，做题时先检查条件，再选择判别法。
```

课程攻略优先记录跨学期仍有价值的信息：

- 课程内容的知识结构与先修要求；
- 不同章节的常见困难和学习方法；
- 教材/参考书的使用方式，而不是上传书本身；
- 考核构成的历史记录，必须标明学期和适用范围；
- 已失效信息的更正记录。

不要写来源不明的“考试重点”、未经核验的考核比例、对教师或同学的人身评价，也不要为了显得完整而猜测课程代码和教材版本。

评价教师时，应聚焦可核验的教学安排和个人学习体验，注明学期，不做人身评价。涉及负面或主观评价时使用教师姓名拼音首字母，并避免让描述可用于骚扰个人。

## 新建一门课程

1. 复制 `templates/course/` 为 `docs/courses/<英文短名>/`；
2. 把模板中的占位文字替换为课程真实信息；
3. 在 `docs/catalog.md` 增加课程；
4. 在 `mkdocs.yml` 的 `nav → 课程目录` 下增加课程名称、课程页与资料清单；
5. 执行本地检查后提交 PR。

英文短名只用于稳定网址，页面标题和文件名可以使用中文。课程名称、代码或院系不确定时请保留“待核验”，不要猜测。

!!! warning "课程不要占用顶部标签"
    所有课程必须作为 `课程目录` 的子项。不要把课程名称添加为 `nav` 的顶层项目，否则课程增多后顶部标签会溢出。

## 管理员首次发布

1. 在 GitHub 创建空仓库，并把本目录推送到 `main` 分支；
2. 修改 `mkdocs.yml`，补上真实的 `site_url` 与 `repo_url`；
3. 打开仓库 **Settings → Pages**，在 **Build and deployment** 中选择 **GitHub Actions**；
4. 推送后查看 **Actions → Deploy course guide**；部署完成后把网站地址加到仓库 About；
5. 在 **Settings → General → Pull Requests** 中启用合并前审核，并为 `main` 配置分支保护（推荐但非必需）。

工作流会在每个 PR 上执行资源登记、严格构建和站内链接检查；只有推送到 `main` 时才发布网站。
